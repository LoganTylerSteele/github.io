An end-to-end quadcopter flight controller — hardware, firmware, and ground software — built under DO-178C–shaped process discipline. The differentiator is not the controller itself; it is the public set of certification-pattern artifacts wrapped around it: PSAC, SDP, SVP, SCMP, SQAP; system / high-level / low-level requirements with full bidirectional traceability; structural coverage instrumentation including selective MC/DC on the four flight-critical decisions; an honest deviation log; a hazard analysis with severity / likelihood matrix; and a STRIDE threat model for the secure C2 link.

Working code includes a bare-metal cooperative scheduler, arming monitor, failsafe manager, mode manager, Mahony attitude estimator, rate PID controller with clamping anti-windup, and an X-quad motor mixer. Each compiles strict-clean under -Wall -Wextra -Werror -Wpedantic -Wconversion -Wshadow; each is unit-tested with explicit MC/DC truth tables where the requirement demands it. A custom Python tool walks the repository, extracts @impl tags and test names, and builds the traceability matrix on every push.

The project pulls together my Avidyne work (DO-178 cert tasks, IBM DOORS, automated C++ testing) and my distributed-systems research (mTLS / ABAC / JWT) into a single artifact set targeted at aerospace / defense recruiters. Apache-2.0 and CERN-OHL-S-2.0 licensed.
